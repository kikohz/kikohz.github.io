Linksys EA6700 刷Advanced Tomato

发表于 2015-10-24 | 阅读次数:

hi，各位1024节快乐哈。

废话不多，开始入正题，EA6700买来有1年多了，刚买那会没有可刷得固件，然后后来也一直没研究，中间有一段时间DD是可以刷了，但是貌似不是太稳定所以也没有去尝试，直到后来无意中发现了Advanced Tomato， 然后查论坛，EA6700 刷Tomato貌似比较稳定，所以就开始尝试刷机。

注:Advanced Tomato没有中文，提别介意的朋友慎重

Advanced Tomato 是国外大神在Shibby Tomato的版本之上写了一个比较现代化的界面：

是不是清爽好多。

目前最新版本是131，另外可刷得机器也比较多：

截图只是一部分，这里给出链接，可以看看自己的路由器是否支持：advancedtomato.com

另外作者还提供了Demo 可以在刷机之前点击看看。

ok，之前的介绍完成，现在正式开始刷机(提前注明一下，由于我是先刷机后写的教程，所以Linksys 官方固件界面我没有做截图，只能去网上找来图片，如有侵权请告知我会修改，谢谢)

刷机之前首先建议做(为了你的路由器不会变砖) Hard Reset 30/30/30

捅RESET之前你先确保DD固件里面的RESET功能是打开了的，一般这个选项是默认打开的！

首先将路由器通电，接着捅RESET孔直到第一个30秒结束，接着在不松掉RESET的前提下，将电源断掉等待30秒，最后将电源在插上去再继续压着RESET孔30秒！就完成这个步骤了！

这里我要说的是整个过程不能松掉RESET孔！也就是说你要按RESET一直90秒！

需要注意的是ASUS的路由器在这个步骤之后一段时间后，一定要拔掉电源再插上电源才能正常启动！

ok做完之后你的路由器就会恢复到出厂状态，然后进入到这个界面:

点击选择固件，在弹出的框中选择你下载的tomato固件，然后系统会让你2次确认，点击确认之后开始刷机，切记刷机过程千万要保证不要断电。

大约5分钟左右会刷好，然后你会搜索到Tomato的无线，密码看你路由器后面的标签(Linksys是有的),也就是出厂默认的那个。

进入刷好的Tomato之后先设置管理员密码和ssh登陆：

设置好之后保存，然后ssh登陆：

ssh root@192.168.1.1

你会看到下面的信息：

剩下的事情就是你自己玩了 。

当然，我们刷机的主要一个目的就是科学安全的上网。

我这里是用Shadowsocks。

首先需要挂载JFFS，按我的截图设置

记得在enable之后执行Fomart，然后继续在Administration ->Scripts中的Init选项卡中输入mount -o bind /jffs/opt /opt 点击Save保存

然后进入ssh里面先把文件夹创建了 cd /jffs mkdir opt

重启路由器。

然后在ssh里面执行mount来看一下挂载

看到箭头所指的说明挂载成功。

接下来我们安装entware，安装(shadowsocks-libev)需要。

执行命令：entware-install.sh。ATT 131版本集成好了entware的安装脚本，大家只要设置好/opt目录的挂载即可直接执行该命令安装entware

entware安装成功之后安装shadowsocks-libev

据网上资料显示entware源里面的版本是存在bug的，所以我自己安装用的是HOUZI自己编译的(最新版本2.4.0) 下载链接在最后

将下载的 shadowsocks-libev_2.4.0-1_armv7soft.ipk 上传到 /opt，安装

root@unknown:/# opkg install shadowsocks-libev_2.4.0-1_armv7soft.ipk

编辑 /opt/etc/shadowsocks.json,这个不多说，懂的人都会

配置好json文件后，

root@unknown:/# /jffs/opt/etc/init.d/S22shadowsocks start
Starting ss-local... done.

说明 shadowsocks 本地代理启动了，可以用 Chrome 浏览器加上 SwitchyOmega 插件测试，

如果ok的话继续下面

配置 ss-redir 透明代理

编辑 /opt/etc/init.d/S22shadowsocks

找到

PROCS=ss-local

改成

PROCS=ss-redir

保存，输入命令

root@unknown:/# /jffs/opt/etc/init.d/S22shadowsocks restart
Starting ss-redir... done.

看到done就说明透明代理启动成功。

ChinaDNS 防污染 和dnsmasq

下载附件包，里面把里面的chinadns放到/opt/bin下。将dnsmasq.d文件夹和shadowsocks放到/jffs目录下，S23ss-tunnel，S24chinadns 放到/jffs/opt/etc/init.d/目录,将所有文件的权限设置为 755

设置是否需要UDP转发：

如果采用UDP转发则不用做任何修改，附件中得文件默认是UDP转发的，如果不用UDP转发打开shadowsocks，屏蔽下面几行：

sed -e "s/shadowsocks.json\"/shadowsocks.json -b 0.0.0.0 -l 5300 -L 8.8.8.8:53 -u\"/" -e "s/PROCS=ss-redir/PROCS=ss-tunnel/" /opt/etc/init.d/S22shadowsocks > /opt/etc/init.d/S23ss-tunnel

sed '5c ARGS="-b 127.0.0.1 -p 5353 -c /opt/etc/chnroute.txt -s 114.114.114.114,127.0.0.1:5300"' /opt/etc/init.d/S24cn > /opt/etc/init.d/S24chinadns

/opt/etc/init.d/S23ss-tunnel start

配置完成后执行以下命令更新 chnroutes 和 chinalist：

/jffs/shadowsocks update
/jffs/shadowsocks update_chinalist

如果配置正确的话以上命令执行后不会报错，且 /jffs 下 ignore.list 里有5000+行ip路由表

配置运行脚本

Advanced Tomato下 Administration -> Scripts -> init 里添加如下命令：

mount -o bind /jffs/opt /opt
/jffs/shadowsocks ipset
/jffs/shadowsocks run

注意顺序不要错！

Administration -> Scripts -> Wan Up 里添加如下命令：

/jffs/shadowsocks rules

Advanced -> DHCP/DNS -> DHCP / DNS Server (LAN) -> Dnsmasq (Custom configuration) 里添加如下命令：

no-poll
no-resolv
all-servers
server=127.0.0.1#5353
conf-dir=/jffs/dnsmasq.d

配置ss-redir维持进程

Administration -> Scheduler -> Custom 1 里添加 /jffs/shadowsocks check ，每分钟检查一次

Administration -> Scheduler -> Custom 2 里添加 echo "" > /var/log/shadowsocks_watchdog.log ，每15分钟清空一次日志

以上配置结束后重启路由器。现在就可以科学上网了
